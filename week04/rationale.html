Rationale
Subject: Home Link Icon

Guideline: Web Style Guide v3 | Page Structure: Site Design

Rationale: I chose to include a home link icon in my page since it's advised that such a link be present in each page of a site. Virtually all major sites have this, including Google, Apple, Microsoft, Facebook, Twitter, Reddit, Instagram, etc. The existence of this icon/logo helps the user know how to get back to the homepage (which, in this case, is Assignments). True, I also have an "Assignments" link button up in the header, but this makes it more obvious how to get to the root page.
Subject: Colors

Guideline: Web Style Guide v3 | Page Design: Visual Design

Rationale: The color schemes of the Assignments page as well as the Old Commerical Site page are distracting at best and horrendous at worst. I figured, this time, to simplify the colors by removing hue and saturation out of virtually everything (except for some links and the company logo). This helps the user feel more comfortable reading the page as there are almost no distracting color combinations.
Frankly, I would add more color in at a future date, but this was the route I chose to go for this week due to time constraints.
Subject: Header Navbar

Guideline: Web Style Guide v3 | Page Structure: Site Design

Rationale: I chose to include a pinned header with links at the top of each page of this week's sub-site in order to give the user a way to quickly find the links to each site's page. I modeled the concept from Apple's website, though on their site the header navigation bar is simply at the top of the page, not pinned. I decided to pin my navigation bar simply out of personal taste and to give the page an extra scrolling feature.
Note that I did not include breadcrumbs like in past weeks. I decided not to this time since all of the necessary links are present in the navigation bar.
Note also that, of course, some pages on this site would still need to be created (namely, "API" and "Register"), so those links are defunct.
Subject: Simplicity

Guideline: Web Style Guide v3 | Interface Design

Rationale: Along the same logic behind the color scheme, I chose to simplify my page design in order to remove the unimpressive, distracting elements. True, I do not feel I had very many unnecessary elements in past weeks, but I did choose to remove things such as round borders in order to present the page in a clear, somewhat more professional way.
That being said, I kept the box-shadows for the divisions as well as added some color transition effects to the link buttons so as to avoid leaving the site too boring.
Subject: Footer

Guideline: Web Style Guide v3 | Page Structure | Site Design

Rationale: I added a footer with some helpful information, including my name and licensing terms. Virtually all major websites have some sort of footer which is necessary for authoring, licensing and copyright reasons. I especially want all users to know that this site is undoubtedly a test site and is in no way intended to be a real business. I did not want this information anywhere else since it's only useful to a select few, so the footer was the best spot to put it.
